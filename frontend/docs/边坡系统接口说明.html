通用约定

项说明
Base URL http://wtapi.jgtop1.com（前端 localStorage baseApi）
鉴权 请求头 Authorization: Bearer <token>。登录成功后 Cookie 也可能写入 EL-ADMIN-TOEKN。
常用头 Accept: application/json； Content-Type: application/json;charset=UTF-8； Origin / Referer: http://bdlx.jgtop1.com

设备类型 type GNSS = 204（TXtData）；雨量 = 201（TResRain）

1. 认证

获取验证码

GET /auth/code 登录必需

返回算术验证码图片（base64）与 uuid，登录时一并提交。

{
  "uuid": "code-key-xxxx",
  "img": "data:image/png;base64,...."
}

登录

POST /auth/login 登录必需 
{
  "username": "15925118626",
  "password": "********",
  "code": "11",
  "uuid": "code-key-xxxx"
}

成功时返回 token（JWT）。后续请求放在 Authorization 头。

当前用户权限

GET /auth/info

返回角色权限列表，如 basicPoint:list 等。

2. 基础台账

监测点列表（含坐标、最新时间）

GET /api/basicPoint 已用于 slope-points.json

返回 { content: [...], totalElements }。本项目关键字段：

字段说明
id点位主键（拉历史数据用）
pointName如 25038476(堆石平台)、雨量
type204=GNSS，201=雨量
lng / lat / alt坐标
lastTime最新数据时间
monitorId / monitorName所属监测对象
sn / iccid 等设备通信信息

字段字典（按设备类型）

GET /api/basicPoint/getDataType 字段释义

返回各设备类型字段元数据。例如 TXtData（卫星定位/GNSS）、 TResRain（雨量：total / yl24h / yl60f / observation 等）。

项目 / 监测对象

GET /api/basicProject
GET /api/basicMonitor
GET /api/basicMonitor/all

本项目监测对象 id=721，项目 id=398。

3. 数据分析（核心）

历史时序数据（GNSS / 雨量）★

POST /api/analysis/getHistoryData 一张图主数据源

说明：对应网页「数据分析 → 基础分析 → 查询」后 Network 中的 getHistoryData。一张图的位移与雨量由此接口生成 data/slope.json。

请求体（实测可用）

{
  "ids": [5759, 5760, 5761],
  "startTime": "2026-06-25 01:03:08",
  "endTime": "2026-07-15 01:03:08"
}
{
  "ids": [5762],
  "startTime": "2026-06-25 01:03:08",
  "endTime": "2026-07-15 01:03:08"
}

ids 为 /api/basicPoint 返回的点位 id；时间格式为 yyyy-MM-dd HH:mm:ss。

响应结构（摘录）

顶层 key 形如 "点位名,类型" 或 "雨量,201,5762"，值为时序数组。

5. 本项目本地接口约定

一张图前端不直连边坡站（避免跨域/验证码），而是定时读本地文件：

GET ./data/slope.json frontend/js/api/slope.js 每 60s 轮询
文件说明
data/slope.json位移 + 雨量（已由 getHistoryData 转换）
data/slope-points.json点位元数据/坐标（来自 basicPoint）
data/raw/getHistoryData.jsonGNSS 原始响应
data/raw/getHistoryData_rain.json雨量原始响应
{
  "updatedAt": "ISO时间",
  "rainfall": {
    "id": "5762",
    "name": "雨量",
    "valueMm": 0.0,
    "cumulativeMm": 171.8,
    "status": "normal",
    "series": [{ "t": "...", "value": 0.4 }]
  },
  "points": [
    {
      "id": "5760",
      "name": "25038476(堆石平台)",
      "x": -4.5, "y": -0.6, "h": -7.6, "plane": 4.5,
      "status": "normal",
      "series": [{ "t": "...", "x": 0, "y": 0, "h": 0, "plane": 0 }]
    }
  ]
}

后期若有后端，只需把 frontend/js/api/slope.js 中的 SLOPE_API.baseUrl 改为真实服务地址，保持上述 DTO 即可。

6. 拉数脚本

脚本路径：frontend/assets/pull_history.py

  • 调用 POST /api/analysis/getHistoryData 分别拉取 GNSS（5759/5760/5761）与雨量（5762）
  • 写入 data/raw/*.json 与 data/slope.json
  • Token 缓存在 frontend/assets/_token.txt（过期后需重新登录写入）
cd frontend/assets
python pull_history.py
